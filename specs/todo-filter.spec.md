# TODOフィルター仕様

## 概要

TODOリストの表示をフィルタリングする機能の振る舞いを定義する。

## データ構造

```typescript
type FilterType = "all" | "active" | "completed";
```

## フィルタータブ

以下の3つのフィルタータブを表示する:

| タブ名 | FilterType | 表示対象 |
|--------|-----------|---------|
| すべて | `"all"` | 全てのTODO |
| 未完了 | `"active"` | `completed === false` のTODO |
| 完了 | `"completed"` | `completed === true` のTODO |

## 振る舞い定義

### フィルター切替

| 条件 | 操作 | 期待結果 |
|------|------|---------|
| 任意のフィルターが選択中 | 別のフィルタータブをクリック | 表示が切り替わる |
| フィルター切替時 | - | 該当するTODOのみがリストに表示される |
| フィルター切替時 | - | 選択中のタブがハイライト（アクティブスタイル）される |

### 初期状態

- ページ読み込み時のデフォルトフィルターは `"all"`（すべて）
- フィルター状態はlocalStorageに保存しない（リロードで「すべて」に戻る）

### 件数表示

| 条件 | 表示 |
|------|------|
| 未完了のTODOがある | 「X件の未完了タスク」と表示 |
| 未完了のTODOがない | 「0件の未完了タスク」と表示 |
| フィルター切替時 | 件数は常に未完了の総数を表示（フィルターに依存しない） |

### エッジケース

| 条件 | 期待結果 |
|------|---------|
| フィルター適用後にTODOがゼロ件 | 「該当するTODOはありません」と表示 |
| TODOが1件もない（初期状態） | 「TODOを追加してください」と表示 |
